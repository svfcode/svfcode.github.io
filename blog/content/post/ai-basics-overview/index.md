### Что это технически? - Огромная статистическая машина {.toc-heading-only}

<details class="post-accordion">
<summary style="cursor: pointer; font-weight: 600;">Что это технически? - Огромная статистическая машина</summary>
<div style="margin-top: 0.75em;">

<p><strong>Определение и принципы работы</strong></p>
<p>Нейросетевые модели, обученные предсказывать следующий токен на основе контекста.</p>
<p>Используют статистические закономерности в языке для генерации текста.</p>

<p>LLM использует полный контекст для определения наиболее вероятного следующего слова.</p>

<p><img src="llm-context.png" alt="Схема: модель опирается на полный контекст при выборе следующего токена" loading="lazy" /></p>

<p><strong>Флоу предсказания токенов LLM</strong></p>
<p><strong>Входной контекст:</strong> «Карл у Клары украл…»</p>
<p><strong>Анализ контекста:</strong></p>
<ul>
<li>Карл (субъект)</li>
<li>у Клары (у кого)</li>
<li>украл (действие)</li>
</ul>
<p><strong>Предсказание:</strong> Модель анализирует все предыдущие токены и их связи, распознаёт паттерн известной скороговорки</p>
<p>→ «кораллы»</p>
<p><strong>Вероятность:</strong> 85%</p>

<p><img src="llm-token-flow.png" alt="Флоу предсказания следующего токена" loading="lazy" /></p>

<p><strong>Трансформеры и архитектура</strong></p>
<p>Механизм self-attention для обработки последовательностей (ну тут всё понятно — на самом деле нет).</p>

<p><img src="attention.png" alt="Механизм self-attention" loading="lazy" /></p>

<p>Параллельная обработка данных вместо рекуррентной.</p>

<p><img src="parallel-vs-rnn.png" alt="Параллельная обработка вместо рекуррентной схемы" loading="lazy" /></p>

<p>Многослойная структура с миллиардами параметров (тепловая карта весов).</p>

<p><img src="layer-weights-heatmap.png" alt="Многослойная структура и масштаб параметров" loading="lazy" /></p>

<p><strong>Обучение на текстовых данных</strong></p>
<ul>
<li>Обучение без учителя на миллиардах текстовых примеров</li>
<li>Предобучение на общих данных с последующей специализацией</li>
<li>Масштабирование данных и вычислительных ресурсов</li>
</ul>

</div>
</details>

### Как использовать? - обзор техник работы с LLM {.toc-heading-only}

<details class="post-accordion">
<summary style="cursor: pointer; font-weight: 600;">Как использовать? - обзор техник работы с LLM</summary>
<div style="margin-top: 0.75em;">

<p><strong>Промпт-инжиниринг</strong></p>
<ul>
<li>Искусство формулирования запросов для лучших результатов</li>
<li>Структурирование инструкций с ролями, примерами и контекстом</li>
<li>Итеративное улучшение запросов для точных ответов</li>
</ul>

<p>Развёрнуто по техникам (zero-shot, few-shot, CoT, роли, step-back и др.): <a class="link" href="/p/ai-basics-prompt-engineering/">Базовый минимум про ИИ – промпт-инжиниринг</a>.</p>

<p><strong>Fine-tuning (дообучение)</strong></p>
<ul>
<li>Адаптация модели под специфические задачи и домены</li>
<li>Использование небольших наборов размеченных данных</li>
<li>RLHF (обучение с подкреплением по обратной связи человека)</li>
</ul>

<p><strong>RAG (Retrieval-Augmented Generation)</strong></p>
<ul>
<li>Расширение знаний модели через поиск в базе данных</li>
<li>Комбинирование внешних источников информации с генерацией</li>
<li>Уменьшение галлюцинаций через опору на проверенные факты</li>
</ul>

<p>Подробнее про этапы, чанкование и типы пайплайнов: <a class="link" href="/p/ai-basics-rag/">Базовый минимум про ИИ – RAG-системы</a>.</p>

<p><strong>Chain-of-thought</strong></p>
<ul>
<li>Пошаговое рассуждение для решения сложных задач</li>
<li>Промежуточные вычисления и логические переходы</li>
<li>Улучшение математических и логических способностей модели</li>
</ul>

### Ключевые понятия {.toc-heading-only}

<details class="post-accordion">
<summary style="cursor: pointer; font-weight: 600;">Ключевые понятия</summary>
<div style="margin-top: 0.75em;">

<p><strong>Токен</strong></p>
<ul>
<li>Минимальная единица текста для обработки моделью</li>
<li>Может быть словом, частью слова или символом</li>
<li>Примеры: «привет» = 1 токен, «непредсказуемость» = 2–3 токена</li>
<li>Токенизация разбивает текст на последовательность токенов</li>
</ul>

<p><strong>Температура</strong></p>
<ul>
<li>Низкая температура (0.1–0.3): предсказуемый, точный текст</li>
<li>Средняя температура (0.7–0.9): баланс креативности и связности</li>
<li>Высокая температура (1.5–2.0): творческий, но менее связный текст
<p style="margin: 0.5rem 0 0 0;"><img src="overview-temperature-high.png" alt="Инфографика: температура генерации LLM — от предсказуемого к разнообразным ответам" style="max-width: 100%; height: auto; border-radius: 8px;" loading="lazy" /></p></li>
</ul>

<p><strong>Типы моделей по назначению</strong></p>
<ul>
<li>Базовые (foundation): предобученные на больших корпусах текста</li>
<li>Инструктивные (instruction-tuned): обучены следовать инструкциям пользователя</li>
<li>Чат-модели: оптимизированы для диалогов и многошаговых разговоров</li>
<li>Специализированные: настроенные под конкретные задачи (код, медицина, юриспруденция)
<p style="margin: 0.5rem 0 0 0;"><img src="overview-model-specialized.png" alt="Схема: от foundation-моделей к специализированным (код, медицина, право)" style="max-width: 100%; height: auto; border-radius: 8px;" loading="lazy" /></p></li>
</ul>

<p><strong>Контекстное окно</strong></p>
<ul>
<li>Ограничение на объём текста, который модель обрабатывает за раз</li>
<li>Различные техники расширения контекста (8K–100K токенов)</li>
<li>Потеря информации при работе с длинными документами</li>
</ul>

</div>
</details>

### Проблемы и ограничения {.toc-heading-only}

<details class="post-accordion">
<summary style="cursor: pointer; font-weight: 600;">Проблемы и ограничения</summary>
<div style="margin-top: 0.75em;">

<p><strong>Галлюцинации</strong></p>
<ul>
<li>Модель может выдумывать несуществующие факты с уверенным видом</li>
<li>Создание правдоподобной, но ложной информации</li>
<li>Сложность верификации сгенерированного контента</li>
</ul>

<p><strong>Вычислительные ресурсы</strong></p>
<ul>
<li>Требуются мощные GPU/TPU для обучения и инференса</li>
<li>Высокие энергозатраты на обучение крупных моделей</li>
<li>Стоимость разработки и поддержки инфраструктуры</li>
</ul>

<p><strong>Этические вопросы</strong></p>
<ul>
<li>Предвзятость и стереотипы в обучающих данных</li>
<li>Проблемы безопасности и генерации вредоносного контента</li>
<li>Нарушение авторских прав и вопросы интеллектуальной собственности</li>
</ul>

</div>
</details>
